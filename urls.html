  • Required version of JAVA -> JDK 1.8
  • If you run as standalone application, do a maven clean build using >mvn install >java -jar airport-info-service-0.0.1-SNAPSHOT.jar
  • 5) Use swagger to use load and airport apis and test it
  • This application is built with spring boot embedded in memory H2 Database.
    Hence, the state is maintained only until the application is up and running. You can change the datasource to external database for persistance.
  • As there are 50000 records of airports, I have used Batch processing to insert the data into in memory database.
    Hence you can trigger http://localhost:8080/load API to load all the data into the database initially.
  • Once the data is loaded onto the DB, you can start using the actual APIs to find Airports by name and Iata code
